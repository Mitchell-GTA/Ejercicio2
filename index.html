¿Qué es EJS?

¿Para qué sirve la "E"? "¿Incrustado?" Podría ser. ¿Qué tal "Eficaz", "Elegante" o simplemente "Fácil"? EJS es un lenguaje de plantillas simple que le permite generar marcado HTML con JavaScript simple. Sin religiosidad sobre cómo organizar las cosas. Sin reinvención de la iteración y el flujo de control. Es simplemente JavaScript.

Utilice JavaScript simple

  • Nos encanta JavaScript. Es un idioma totalmente amigable. Todos los lenguajes de plantillas llegan a ser Turing completos. ¡Simplemente elimine el intermediario y use JS!

Tiempo de desarrollo rápido

  • No pierda tiempo y atención averiguando una nueva sintaxis arcana debido a la 'elegancia', o cómo preprocesar sus datos para que realmente se procesen correctamente.

Sintaxis simple

  • Código JavaScript en etiquetas scriptlet sencillas y directas. ¡Simplemente escriba JavaScript que emita el HTML que desea y haga el trabajo!

Fácil depuración

  • Es fácil depurar errores de EJS: sus errores son excepciones simples de JavaScript, con números de línea de plantilla incluidos.

Desarrollo activo

  • EJS tiene una gran comunidad de usuarios activos y la biblioteca está en desarrollo activo. Nos complace responder a sus preguntas o brindarle ayuda.

Caracteristicas

  • Rápida compilación y renderizado
  • Etiquetas de plantilla simples: <%%>
  • Delimitadores personalizados (p. Ej., Utilice [??] En lugar de <%%>)
  • La sub-plantilla incluye
  • Se envía con CLI
  • Soporte para navegador y servidor JS
  • Almacenamiento en caché estático de JavaScript intermedio
  • Almacenamiento en caché estático de plantillas
  • Cumple con el sistema Express View

Instalar en pc

Es fácil instalar EJS con NPM.

$ npm install ejs

Utilizar

Pase a EJS una cadena de plantilla y algunos datos. BOOM, tienes algo de HTML.

let ejs = require('ejs'); let people = ['geddy', 'neil', 'alex']; let html = ejs.render('<%= people.join(", "); %>', {people: people});

Opciones

  • filenameUsado por cachepara claves de caché y para incluye
  • rootEstablezca la raíz del proyecto para las inclusiones con una ruta absoluta (por ejemplo, /file.ejs). Puede ser una matriz para intentar resolver la inclusión de varios directorios.
  • views Una matriz de rutas para usar al resolver inclusiones con rutas relativas.
  • context Contexto de ejecución de la función
  • compileDebugCuando falseno se compila la instrumentación de depuración
  • client Devuelve la función compilada independiente
  • delimiter
  • openDelimiter
  • closeDelimiter '/ li>
  • debug Cuerpo de función generado de salidas
  • strict Cuando se establece en "verdadero", la función generada está en modo estricto
  • _withSi usar o no with() {}constructos. Si falseentonces los locales se almacenarán en el localsobjeto. (Implica `- estricto`)
  • localsNameNombre que se utilizará para el objeto que almacena variables locales cuando no se utilice withValores predeterminados paralocals
  • rmWhitespaceElimine todos los espacios en blanco seguros para eliminar, incluidos los espacios en blanco iniciales y finales. También permite una versión más segura de -%>slurping de línea para todas las etiquetas scriptlet (no elimina las nuevas líneas de etiquetas en el medio de una línea).
  • escapeLa función de escape utilizada con <%=construct. Se utiliza en el renderizado y se .toString()edita en la generación de funciones de cliente. (De forma predeterminada, escapa a XML).
  • outputFunctionNameEstablezca una cadena (por ejemplo, 'echo'o 'print') para que una función imprima la salida dentro de las etiquetas scriptlet.
  • asyncCuando true, EJS usará una función asíncrona para renderizar. (Depende del soporte async / await en el tiempo de ejecución de JS.

Etiquetas

  • <% Etiqueta 'Scriptlet', para control de flujo, sin salida
  • <%_ Etiqueta Scriptlet 'Whitespace Slurping', elimina todos los espacios en blanco antes de ella
  • <%= Envía el valor a la plantilla (HTML escapado)
  • <%- Envía el valor sin escape a la plantilla
  • <%# Etiqueta de comentario, sin ejecución, sin salida
  • <%% Genera un literal '<%'
  • %> Etiqueta final simple
  • -%> Etiqueta de modo de recorte ('newline slurp'), recortes después de la nueva línea
  • _%> Etiqueta final 'Whitespace Slurping', elimina todos los espacios en blanco después de ella

Incluye

Las inclusiones son relativas a la plantilla con la includeconvocatoria. (Esto requiere la opción 'nombre de archivo'). Por ejemplo, si tiene "./views/users.ejs" y "./views/user/show.ejs", usaría <%- include('user/show'); %>.

Es probable que desee utilizar la etiqueta de salida sin formato ( <%-) con su inclusión para evitar el doble escape de la salida HTML.

<ul>     % users.forEach(function(user){ %>         %- include('user/show', {user: user}); >     % }); %> </ul
Diseños

EJS no admite bloques específicamente, pero los diseños se pueden implementar al incluir encabezados y pies de página, así:

<%- include('header'); -%> <h1>     Title </h1> <p>     My page </p> <%- include('footer'); -%>
